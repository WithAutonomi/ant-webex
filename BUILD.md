# Build Instructions (AMO Source Code Submission)

These instructions let a reviewer reproduce an **exact, byte-for-byte copy** of the
add-on package from source.

The extension is written in **TypeScript**. The human-authored source lives in `src/`.
The build step (esbuild) bundles and minifies it into the `dist-firefox/` directory,
which is what gets packaged and uploaded to AMO. None of the files in `src/` are
transpiled, concatenated, minified, or machine-generated — they are the original
source.

## 1. Operating system and build environment

The build is cross-platform and has no native compilation step. It runs on:

- **Linux** (any modern distro)
- **macOS** (12+)
- **Windows** (10/11)

No OS-specific tools are required. The reference build for the uploaded package was
produced on **Windows 11**, but the output is identical on any supported OS because
the only build tool (esbuild) is a version-pinned binary selected by the lockfile.

## 2. Required software and versions

| Program | Version used | Minimum | Install |
|---------|--------------|---------|---------|
| Node.js | 24.15.0 | 18.0 (any LTS) | https://nodejs.org/en/download — download the LTS installer for your OS and run it. |
| npm | 11.12.1 | bundled with Node | Installed automatically by the Node.js installer. |

No other global tools are needed. The exact versions of the two build dependencies
(**esbuild 0.25.12** and **typescript 5.9.3**) are pinned in `package-lock.json` and
installed locally by `npm ci` — they do **not** depend on the Node version, which is
why the output is reproducible.

Verify your toolchain:

```bash
node --version    # v18.x or newer
npm --version
```

## 3. Build script

The build is fully automated by `build.mjs` (invoked via the `build:firefox` npm
script). It performs every step needed: bundling the TypeScript entry points with
esbuild, merging the base manifest with the Firefox manifest overlay, and copying the
static HTML/CSS/icon assets into the output directory.

## 4. Step-by-step build

Run these commands from the root of the unzipped source folder:

```bash
# 1. Install the exact, pinned build dependencies from package-lock.json
npm ci

# 2. Build the Firefox package
npm run build:firefox
```

This produces the **`dist-firefox/`** directory. Its contents are the unpacked
add-on and match the uploaded package exactly:

```
dist-firefox/
├── manifest.json
├── background.js
├── content.js
├── assets/icons/icon-16.png
├── assets/icons/icon-32.png
├── assets/icons/icon-48.png
├── assets/icons/icon-96.png
├── assets/icons/icon-128.png
├── onboarding/index.html
├── onboarding/index.js
├── onboarding/style.css
├── popup/index.html
├── popup/index.js
└── popup/style.css
```

The build is **deterministic**: running `npm ci && npm run build:firefox` on a clean
checkout always produces identical file hashes.

### (Optional) Verify against the uploaded package

The uploaded `.xpi` is a ZIP of the **contents** of `dist-firefox/` (the files at the
top level of the archive, not the `dist-firefox` folder itself). To compare, unzip the
uploaded `.xpi` and diff it against the freshly built `dist-firefox/`.

### (Optional) Type-check

```bash
npm run typecheck   # tsc --noEmit, no output produced
```

## 5. What is in this source submission

Included (everything needed to build):

- `src/` — all human-authored TypeScript, HTML, CSS, and manifest source files
- `assets/` — extension icons
- `build.mjs` — the build script
- `package.json`, `package-lock.json` — dependency manifest with pinned versions
- `tsconfig.json` — TypeScript configuration
- `README.md`, `ARCHITECTURE.md`, `BUILD.md` — documentation

Excluded (not source; regenerated by the build, per `.gitignore`):

- `node_modules/` — restored by `npm ci`
- `dist/`, `dist-firefox/` — build output
